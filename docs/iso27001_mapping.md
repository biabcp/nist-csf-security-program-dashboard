# ISO 27001 Light Crosswalk (NIST CSF Program)

## Scope and Intent
This is a lightweight, practical mapping between the NIST CSF-aligned control catalog and ISO/IEC 27001:2022 Annex A themes. It is intended for portfolio alignment, not certification evidence by itself.

## Crosswalk Highlights
| NIST CSF Theme | Example Controls | ISO 27001 Reference (Light) | Notes |
|---|---|---|---|
| Asset Management & Context | ID-01, ID-02, ID-04 | A.5.9, A.5.30, A.8.9 | Supports scope, inventory, and business context requirements. |
| Governance & Policy | ID-06 | A.5.1 | Policy lifecycle and governance operating model. |
| Supplier Security | ID-05 | A.5.19 | Third-party risk and contractual security obligations. |
| Identity & Access | PR-01, PR-02, PR-03 | A.5.16, A.8.2, A.8.5 | Workforce and privileged identity controls. |
| Data Protection | PR-08, PR-09 | A.8.24 | Encryption and key management practices. |
| Secure Engineering | PR-10 | A.8.28 | Secure coding and security testing integration. |
| Logging & Monitoring | DE-01, DE-04, DE-06 | A.8.15, A.8.16 | Monitoring pipeline and detection readiness. |
| Incident Management | RS-01, RS-02, RS-04 | A.5.24, A.5.26, A.5.31 | Response process and communication obligations. |
| Resilience & Recovery | RC-01, RC-02, RC-04 | A.5.30, A.5.27 | Business continuity and corrective-action closure. |

## Practical Guidance
- Use this crosswalk to answer customer questionnaires that request ISO alignment while operating primarily on NIST CSF.
- For certification readiness, map controls to Statement of Applicability (SoA) and gather control-specific evidence packs.
- Maintain one control implementation record and reference it across multiple frameworks to reduce duplicate work.
